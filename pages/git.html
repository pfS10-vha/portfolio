Compétence Technique - Git

Définition

    Git est ce que l'on appelle un « système de contrôle de version », il va permettre à chaque développeur de travailler sur un même projet tout en gardant un historique des modifications apportées au code par chacun. Cela permet, en cas de suppression accidentelle de code, de revenir en arrière. C'est ce que l'on appelle le « versionning ». Il n'est toutefois pas seulement utilisé par les développeurs, il est également utile pour les administrateurs système afin de garder une trace de leurs scripts et documentations. Git est ainsi devenu un incontournable dans le monde professionnel afin de faciliter le travail en équipe.

Éléments de preuve

    Au cours de mes projets, j'ai essentiellement utilisé Git afin de partager des scripts et des fichiers de configuration avec mes collègues. Notamment dans le cadre d'un projet d'étude, nous avons très souvent tendances à réaliser des scripts qui sont par la suite exécutés sur différentes machines et il arrive parfois que nous nous retrouvions avec plusieurs versions différentes du script et que nous ne sachions pas laquelle est la dernière version fonctionnelle. Pour remédier à cela, j'ai mis en place une tâche journalière qui permet d'envoyer le script dans un répertoire Git lorsqu'il y a eu un changement et de récupérer la dernière version de ce même script sur la machine. Cela nous permet, dans un premier temps, de synchroniser les scripts sur toutes nos machines, d'avoir un historique des versions, et aussi, de centraliser l'endroit où sont sauvegardés nos scripts.

    Durant plusieurs autres projets d'études, nous avons eu l'occasion de travailler en équipe et par conséquent d'utiliser Git. Par exemple, lors de ma première année, nous devions, en équipe de trois personnes, réaliser un site internet pour une petite boutique. Étant un projet à réaliser en équipe, le choix d'utiliser Git nous a paru évident. Toutefois, centraliser notre travail n'a pas été le seul objectif. En effet, notre commanditaire pouvait parfois être un peu indécis et nous demander de revenir en arrière sur une fonctionnalité. C'est là où le versionning de Git s'est avéré utile, puisque nous avions la possibilité de facilement revenir en arrière lorsqu'il y en avait besoin. Par ailleurs, le fait de centraliser notre code nous a également permis de partager le lien d'accès au Git avec notre commanditaire afin qu'il puisse suivre l'avancée des travaux et tester en direct les modifications que nous effectuions.

Autocritique

    Bien que je maîtrise des fonctionnalités de base de Git telles que le clone, le commit, le push, le pull ainsi que le concept de branch, je suis conscient que mes compétences restent plutôt limitées par rapport à une utilisation plus avancée de cette technologie. Notamment le concept de Continous Integration/Continous Deployment (CI/CD) qui reste pour moi encore un peu vague. Le CI/CD est en fait une façon de configurer et d'utiliser Git qui permet de valider et d'automatiser les livraisons du projet au client. Je n'ai pas encore eu l'opportunité de le mettre en place puisque je n'ai, jusqu'à maintenant, utilisé Git que pour des petits projets, mais le CI/CD est un concept très important dans le cadre professionnel lorsqu'il y a un attendu client.

    Par conséquent, je conseille fortement la maîtrise et la compréhension de l'outil Git, c'est un incontournable dans n'importe quel projet informatique et est même utile pour un petit projet personnel afin de garder une trace des modifications effectuées.

Évolution

    Enfin, comme précédemment expliqué, j'aimerais apprendre à mettre en place un système CI/CD pour en comprendre le fonctionnement et atteindre un niveau de maîtrise qui pourrait me permettre de réaliser et de livrer moi-même mes propres projets personnels. Afin d'atteindre cet objectif, j'aimerais tout d'abord suivre une formation reprenant quelques bases du fonctionnement de Git ainsi qu'une formation plus poussée et centrée sur le CI/CD.
